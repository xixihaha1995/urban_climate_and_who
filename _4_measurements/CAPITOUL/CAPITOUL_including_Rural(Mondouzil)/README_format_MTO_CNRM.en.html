CAPITOUL 2004-2005

Description of data files "mesures météorologiques classiques des stations mises en place par l'équipe GMEI/4M du CNRM" for the CAPITOUL Data Base.

1- General description :

Files are text files, containing one line/instant and one column/parameter.
Each file contains the data of a site during a month between january 2004 and february 2005 (so 14 files by site), except  for the site of Saint-Sardos, on which measurements only began on april 6th, 2004 (only 11 files).

The site of measurements (and acronyms + geographic position) are : 
  • Auzeville : acronym : Az (lon : 1.5049   lat : 43.5291)
  • Launac : acronym : La (lon : 1.??????   lat : 43.???????)
  • Mondouzil : acronym : Mo (lon : 1.5699   lat : 43.6286)
  • Toulouse-Météo : acronym : Tm (lon : 1.3740   lat : 43.5747)
  • Toulouse-GDF : acronym : Tg (lon : 1.??????   lat : 43.??????)
  • Le-Fauga : acronym : fa (lon : 1.2922   lat : 43.3853)
  • Saint-Sardos : acronym : Cs (lon : 1.??????   lat : 43.??????)
File names have the naming convention : SS_BDD_mm-yyyy.asc , where
  • SS is the acronym of each measurement site (Az --> Auzeville)
  • mm is the month
  • yyyy is the year(2004 or 2005).

2- Data file contents :

Each file begins with a descriptive header : 15 lines :
  • parameter names,
  • parameter units,
  • code for missing values,
  • references of sensors : 
    • If there is no sensor, reference is "No_sensor".
    • If the parameter is resulting of an operation with 2 (or more) measurements, the references of all sensors are precised : "sensor1 AND sensor2"
All files contain the same number of data column, even if a peculiar sensor was not mounted (for file format homogeneity reasons) :
    - if a parameter isn't measured, the column also contains the code "+9999"
        and the sensor name is "No_sensor" (asq ais in the header description above).
    - if sensor is present, but reporting no (or erroneous) measurement, le code is generally "-32000" (exact value in the header of ascii file)

14 columns, separator: space
  1. date, format DD/MM/AAAA
  2. time, format HHMNSS.SSS
  3. air temperature (position 2 meters high) : Celsius
  4. air pressure: hPa
  5. relative humidity (position 2 meters high) : %
  6. rain rate: mm/h
  7. wind direction (position 10 meters high) : degree
  8. wind speed (position 10 meters high) : m/s
  9. incoming short wave radiation: watts/m2
  10. outgoing short wave radiation: watts/m2
  11. incoming long wave-Red radiation: watts/m2
  12. outgoing long wave radiation: watts/m2
  13. air temperature (position 6 meters high) : Celsius
  14. air relative humidity (position 6 meters high) : Celsius

3- Data frequency :

The sampling rate is the minute for each measurement site, only Le-Fauga has a lower data frequency (30 minutes).
  • Auzeville : sampling rate 1 minute (measurements 10 sec, averages by automatic station --> minute)
  • Launac :  sampling rate 1 minute (measurements 10 sec, averages by automatic station --> minute)
  • Mondouzil : sampling rate 1 minute (measurements 10 sec, averages by automatic station --> minute)
  • Toulouse-Météo : sampling rate 1 minute (measurements 10 sec, averages by automatic station --> minute)
  • Toulouse-GDF :  sampling rate 1 minute (measurements 10 sec, averages by automatic station --> minute)
  • Le-Fauga :  sampling rate 30 minutes (measurements minute, averages by automatic station --> 30 minutes)
  • Saint-Sardos : 2 periods :
    • -06 april 2004 --> 6 january 2005 : (measurement and aquisition every 10 sec, averaged to 1 by post-processing).
    • -after 6th january 2005 : sampling rate 1 minute (measurements 10 sec, averages by automatic station --> minute)
