CAPITOUL 2004-2005

1- Measurement's Platform
         
          Ta : Toulouse-Alsace (Tower on last flat floor of 5 levels' building)
Cs : Saint-Sardos (the measurements  are positionned 2 m high from soil)
fa  : Le Fauga (the measurements are on 2 levels)

2- Files
Names of the files:

Ab_BDD_FLUX_MM-YYYY.asc

Files'names are builded like that below :

Ab : Platform's Acronym name (example : Cs for Saint-Sardos...)
MM-YYYY : Date with :
MM : Month in 2 digits
YYYY : Year in 4 digits

.asc : means that it is a "text" file (ascii format)

For each platform, there are 14  files from january 2004 until february 2005 (end of campaign).

3- What you can find inside the files already present on CAPITOUL database :

Each file begins with a descriptive header : 15 lines :
First we can read measurement's information : starting and ending hour, date, platform, (aso..).
End of header is indicated with #END OF HEADER#
Next come 2 lines about parameters : name, unit, and non-present indicator (non computed, non realist, not here),
ending with #END OF PARAMETER LIST#

All files contain the same number of data column, even if a peculiar sensor was not mounted (for file format homogeneity reasons) :
    - if a parameter isn't measured, the column also contains the code "+9999"
        and the sensor name is "No_sensor" (asq ais in the header description above).
    - if sensor is present, but reporting no (or erroneous) measurement, le code is generally "-32000" (exact value in the header of ascii file)
